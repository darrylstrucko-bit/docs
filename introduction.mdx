DSS Stringers is an automated documentation platform built for Node.js teams. Connect your GitHub repository once, and DSS Stringers takes care of the rest — analysing your code on every push, generating accurate and up-to-date documentation with AI, and publishing it to your Mintlify site without any manual intervention. Whether you deploy through Vercel or push directly to GitHub, your docs stay in sync with your code.

## Get up and running

Follow these four steps to go from zero to living documentation for your project.

<Steps>
  <Step title="Create your DSS Stringers account">
    Sign up at [app.dssstringers.com](https://app.dssstringers.com) and create your first project. You will receive an API key that identifies your project in the pipeline.
  </Step>
  <Step title="Connect your GitHub repository">
    Install the DSS Stringers GitHub App on the repo you want to document. This grants DSS Stringers the read access it needs to analyse your code on every push.
  </Step>
  <Step title="Configure and push">
    Add your API key as a repository secret, drop an optional `.dssstringers.json` config file into your project root, and push a commit to trigger your first doc generation run.
  </Step>
  <Step title="View your published docs">
    Open your Mintlify site — your documentation is live and will update automatically on every subsequent deploy. No manual editing required.
  </Step>
</Steps>

<Tip>
  If you are already using Vercel for deployments, DSS Stringers can hook directly into your Vercel pipeline so doc generation fires on every successful production deploy — no extra CI configuration needed. See the [How It Works](/how-it-works) page for details.
</Tip>
