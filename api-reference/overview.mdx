The DSS Stringers REST API gives you programmatic access to your projects, generated documentation, sync jobs, and webhooks. All communication happens over HTTPS, and every response body is JSON. Use this reference alongside the endpoint pages to integrate DSS Stringers into your own tooling, CI pipelines, or internal dashboards.

## Base URL

All API endpoints are relative to the following base URL:

```
https://api.dssstringers.com/v1
```

## Authentication

Every request must include your API key in the `Authorization` header as a Bearer token.

```
Authorization: Bearer YOUR_API_KEY
```

See the [Authentication](/api-reference/authentication) page for full details on finding, using, and rotating your key.

## Request & Response Format

Set `Content-Type: application/json` on every request that includes a body. All response bodies are also JSON.

<CodeGroup>

```bash cURL
curl https://api.dssstringers.com/v1/projects \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json"
```

```javascript Node.js
const response = await fetch('https://api.dssstringers.com/v1/projects', {
  headers: {
    Authorization: `Bearer ${process.env.DSSS_API_KEY}`,
    'Content-Type': 'application/json',
  },
});

const data = await response.json();
console.log(data);
```

</CodeGroup>

## Rate Limits

The API enforces a limit of **60 requests per minute** per API key. If you exceed this threshold, the API returns a `429` status code with the `rate_limit_exceeded` error code. Implement exponential backoff and retry logic in your client to handle this gracefully.

## Timestamps

All timestamps in requests and responses are formatted as **ISO 8601 UTC strings**, for example `2024-01-15T10:30:00Z`. When filtering by timestamp query parameters, supply values in the same format.

## Pagination

List endpoints support cursor-free, page-based pagination via two query parameters:

<ParamField query="page" type="integer" default="1">
  The page number to retrieve.
</ParamField>

<ParamField query="limit" type="integer" default="20">
  The number of items to return per page. Maximum is `100`.
</ParamField>

Every paginated response includes a `meta` object alongside the data array:

<ResponseField name="meta" type="object">
  Pagination metadata for the current response.
  <Expandable title="meta fields">
    <ResponseField name="meta.total" type="integer">
      Total number of items across all pages.
    </ResponseField>
    <ResponseField name="meta.page" type="integer">
      The current page number.
    </ResponseField>
    <ResponseField name="meta.limit" type="integer">
      The number of items returned per page.
    </ResponseField>
  </Expandable>
</ResponseField>

**Example paginated response:**

```json
{
  "data": [...],
  "meta": {
    "total": 84,
    "page": 2,
    "limit": 20
  }
}
```

## Error Format

When a request fails, the API returns a JSON object with a single `error` key containing a machine-readable `code` and a human-readable `message`.

```json
{
  "error": {
    "code": "invalid_api_key",
    "message": "The provided API key is invalid or expired."
  }
}
```

### Error Codes

| Code | HTTP Status | Description |
|---|---|---|
| `invalid_api_key` | 401 | The API key is missing, malformed, or expired. |
| `not_found` | 404 | The requested resource does not exist. |
| `rate_limit_exceeded` | 429 | You have sent more than 60 requests in the current minute. |
| `validation_error` | 422 | One or more request body fields failed validation. |
| `internal_error` | 500 | An unexpected error occurred on the server. |
