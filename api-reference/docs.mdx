Once DSS Stringers has synced a project, it stores each generated page as structured MDX content that you can read back through the API. Use the Docs endpoints to pull page content into your own systems, compare revisions, or build custom documentation UIs. The Syncs endpoints let you audit the history of every sync job and investigate failures.

---

## GET /docs/:projectId

Retrieve all generated documentation pages for a project. Supports [pagination](/api-reference/overview#pagination) and optional timestamp filtering so you can efficiently poll for changes.

### Path Parameters

<ParamField path="projectId" type="string" required>
  The `id` or `slug` of the project whose docs you want to retrieve.
</ParamField>

### Query Parameters

<ParamField query="page" type="integer" default="1">
  Page number to retrieve.
</ParamField>

<ParamField query="limit" type="integer" default="20">
  Number of pages per response. Maximum `100`.
</ParamField>

<ParamField query="updated_after" type="string">
  An ISO 8601 UTC timestamp. When provided, only pages updated **after** this time are returned. Useful for incremental polling (e.g. `2024-01-15T10:00:00Z`).
</ParamField>

### Response — 200 OK

<ResponseField name="data" type="array">
  Array of doc page objects.
  <Expandable title="doc page fields">
    <ResponseField name="data[].id" type="string">
      Unique identifier for the doc page.
    </ResponseField>
    <ResponseField name="data[].title" type="string">
      The page title as extracted or generated by DSS Stringers.
    </ResponseField>
    <ResponseField name="data[].slug" type="string">
      URL-safe slug for the page (e.g. `"getting-started"`).
    </ResponseField>
    <ResponseField name="data[].content" type="string">
      Full MDX content string for the page.
    </ResponseField>
    <ResponseField name="data[].updated_at" type="string">
      ISO 8601 UTC timestamp of the last time this page was regenerated.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="meta" type="object">
  <Expandable title="meta fields">
    <ResponseField name="meta.total" type="integer">Total pages across all results.</ResponseField>
    <ResponseField name="meta.page" type="integer">Current page number.</ResponseField>
    <ResponseField name="meta.limit" type="integer">Items per page.</ResponseField>
  </Expandable>
</ResponseField>

<CodeGroup>

```bash cURL
curl "https://api.dssstringers.com/v1/docs/proj_abc123?page=1&limit=20" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```javascript Node.js
const response = await fetch(
  'https://api.dssstringers.com/v1/docs/proj_abc123?page=1&limit=20',
  {
    headers: {
      Authorization: `Bearer ${process.env.DSSS_API_KEY}`,
    },
  }
);

const { data, meta } = await response.json();
console.log(`Retrieved ${data.length} of ${meta.total} pages`);
```

```json Response
    {
      "id": "page_002",
```

## GET /docs/:projectId/:pageSlug

Retrieve a single documentation page by its slug.

### Path Parameters

<ParamField path="projectId" type="string" required>
  The `id` or `slug` of the project.
</ParamField>

<ParamField path="pageSlug" type="string" required>
  The slug of the doc page to retrieve (e.g. `"getting-started"`).
</ParamField>

### Response — 200 OK

Returns a single doc page object with the same fields described above.

<ResponseField name="id" type="string">Unique identifier for the doc page.</ResponseField>
<ResponseField name="title" type="string">The page title.</ResponseField>
<ResponseField name="slug" type="string">URL-safe page slug.</ResponseField>
<ResponseField name="content" type="string">Full MDX content string for the page.</ResponseField>
<ResponseField name="updated_at" type="string">ISO 8601 UTC timestamp of the last regeneration.</ResponseField>

<CodeGroup>

```bash cURL
curl "https://api.dssstringers.com/v1/docs/proj_abc123/getting-started" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```javascript Node.js
const response = await fetch(
  'https://api.dssstringers.com/v1/docs/proj_abc123/getting-started',
  {
    headers: {
      Authorization: `Bearer ${process.env.DSSS_API_KEY}`,
    },
  }
);

const page = await response.json();
console.log(page.title, page.slug);
```

```json Response
{
  "id": "page_001",
  "title": "Getting Started",
  "slug": "getting-started",
  "content": "---\ntitle: Getting Started\n---\n\nInstall the SDK with `npm install @dssstringers/sdk`...",
  "updated_at": "2024-01-15T10:30:00Z"
```

</CodeGroup>

## GET /projects/:id/syncs

List the sync job history for a project, sorted by most recent first.

### Path Parameters

<ParamField path="id" type="string" required>
  The `id` or `slug` of the project.
</ParamField>

### Query Parameters

<ParamField query="page" type="integer" default="1">
  Page number to retrieve.
</ParamField>

<ParamField query="limit" type="integer" default="20">
  Number of sync records per page. Maximum `100`.
</ParamField>

### Response — 200 OK

<ResponseField name="data" type="array">
  Array of sync job objects.
  <Expandable title="sync job fields">
    <ResponseField name="data[].id" type="string">
      Unique identifier for the sync job (e.g. `"sync_xyz789"`).
    </ResponseField>
    <ResponseField name="data[].status" type="string">
      Current status of the sync: `queued`, `running`, `success`, or `failed`.
    </ResponseField>
    <ResponseField name="data[].started_at" type="string">
      ISO 8601 UTC timestamp of when the job was queued.
    </ResponseField>
    <ResponseField name="data[].completed_at" type="string">
      ISO 8601 UTC timestamp of when the job finished. `null` if still in progress.
    </ResponseField>
    <ResponseField name="data[].pages_updated" type="integer">
      Number of documentation pages created or updated during the sync. `null` if the job has not completed.
    </ResponseField>
  </Expandable>
</ResponseField>

<CodeGroup>

```bash cURL
curl "https://api.dssstringers.com/v1/projects/proj_abc123/syncs?page=1&limit=20" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```javascript Node.js
const response = await fetch(
  'https://api.dssstringers.com/v1/projects/proj_abc123/syncs?page=1&limit=20',
  {
    headers: {
      Authorization: `Bearer ${process.env.DSSS_API_KEY}`,
    },
  }
);

const { data, meta } = await response.json();
console.log(`${meta.total} syncs total; most recent status: ${data[0]?.status}`);
```

```json Response
{
  "data": [
    {
      "id": "sync_xyz789",
      "status": "success",
      "started_at": "2024-01-15T10:30:00Z",
      "completed_at": "2024-01-15T10:31:45Z",
      "pages_updated": 12
    },
    {
      "id": "sync_xyz788",
      "status": "failed",
      "started_at": "2024-01-14T08:00:00Z",
      "completed_at": "2024-01-14T08:00:22Z",
      "pages_updated": null
    }
  ],
  "meta": {
    "total": 38,
    "page": 1,
    "limit": 20
  }
}
```

</CodeGroup>

## GET /projects/:id/syncs/:syncId

Get the full details for a specific sync job, including any error message generated during a failure.

### Path Parameters

<ParamField path="id" type="string" required>
  The `id` or `slug` of the project.
</ParamField>

<ParamField path="syncId" type="string" required>
  The unique identifier of the sync job to inspect.
</ParamField>

### Response — 200 OK

Returns a single sync object with an additional `error` field for failed jobs.

<ResponseField name="id" type="string">Unique sync job identifier.</ResponseField>
<ResponseField name="status" type="string">Job status: `queued`, `running`, `success`, or `failed`.</ResponseField>
<ResponseField name="started_at" type="string">ISO 8601 UTC timestamp when the job was queued.</ResponseField>
<ResponseField name="completed_at" type="string">ISO 8601 UTC timestamp when the job finished. `null` if still in progress.</ResponseField>
<ResponseField name="pages_updated" type="integer">Number of pages created or updated. `null` if not yet completed.</ResponseField>
<ResponseField name="error" type="string | null">
  Human-readable error message when `status` is `"failed"`. `null` for all other statuses.
</ResponseField>

<CodeGroup>

```bash cURL
curl "https://api.dssstringers.com/v1/projects/proj_abc123/syncs/sync_xyz788" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```javascript Node.js
const response = await fetch(
  'https://api.dssstringers.com/v1/projects/proj_abc123/syncs/sync_xyz788',
  {
    headers: {
      Authorization: `Bearer ${process.env.DSSS_API_KEY}`,
    },
  }
);

const sync = await response.json();
if (sync.status === 'failed') {
  console.error('Sync failed:', sync.error);
}
```

```json Response
{
  "id": "sync_xyz788",
  "status": "failed",
  "started_at": "2024-01-14T08:00:00Z",
  "completed_at": "2024-01-14T08:00:22Z",
  "pages_updated": null,
  "error": "Unable to clone repository: authentication failed. Check that your repo URL is correct and the integration has read access."
}
```

</CodeGroup>
